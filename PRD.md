# ⚡ 레이저 서바이벌 — 기획서

## 1. 개요

**프로젝트명:** 레이저 서바이벌 (Laser Survival)
**장르:** 아케이드 서바이벌
**플랫폼:** 웹 브라우저 (데스크톱/모바일)
**기술 스택:** HTML5, Vanilla JavaScript, Canvas API, IndexedDB
**레퍼런스:** [death.fun/laser](https://death.fun/laser)
**컨셉:** 그리드 위에서 방향키로 말을 조작하며 레이저를 피해 생존하는 타임 서바이벌 게임

---

## 2. 핵심 규칙

1. 8×8 그리드 위에 플레이어(말)가 존재한다
2. 일정 간격으로 레이저가 행 또는 열 한 줄을 제거한다
3. 플레이어는 방향키(←↑↓→)로 매 틱마다 한 칸씩 이동한다
4. 레이저에 맞으면 게임 오버
5. 시간이 지날수록 레이저 발사 간격이 짧아진다
6. 60초 생존 시 승리

---

## 3. 사용 HTML5 기술

| 기술 | 역할 | 상세 |
|------|------|------|
| **Canvas API** | 게임 렌더링 | 그리드, 플레이어, 레이저 빔, 파티클, 화면 흔들림 등 모든 비주얼 |
| **IndexedDB** | 기록 저장 | 최고 생존 시간, 플레이 횟수, 승리 횟수 로컬 영구 저장 |
| **Keyboard Event** | 조작 | 방향키 입력으로 플레이어 이동 |
| **requestAnimationFrame** | 게임 루프 | 60fps 렌더링 루프 |

## 5. 그리드 시스템

### 5.1 기본 구조

- 8×8 격자 (총 64칸)
- 각 셀은 정사각형, 간격(gap)으로 구분
- 제거된 칸은 빈 공간(구멍)으로 변환
- 플레이어는 구멍 위로 이동 불가

### 5.2 그리드 상태

| 상태 | 시각 표현 |
|------|----------|
| 활성 | 밝은 셀 |
| 경고 (레이저 예고) | 붉은 깜빡임 |
| 제거됨 | 빈 공간 (어두운 구멍) |
| 플레이어 위치 | 발광 마커 |

---

## 6. 레이저 시스템

### 6.1 발사 규칙

- 첫 발사: 게임 시작 3초 후
- 매 발사 후 다음 발사까지의 간격이 점점 짧아짐
- 발사 1초 전 해당 행/열이 경고 색으로 깜빡임

### 6.2 속도 커브

| 경과 시간 | 레이저 간격 | 체감 |
|-----------|------------|------|
| 0~10초 | 2.8초 | 여유로움 |
| 10~20초 | 2.1초 | 보통 |
| 20~30초 | 1.5초 | 긴장 |
| 30~40초 | 1.1초 | 빠름 |
| 40~50초 | 0.8초 | 매우 빠름 |
| 50~60초 | 0.6초 | 극한 |

공식: `다음 간격 = 현재 간격 × 0.92` (최소 0.6초)

### 6.3 레이저 타겟 선택

- 행(가로) 또는 열(세로) 중 랜덤 선택
- 플레이어가 위치한 줄은 제외 (즉사 방지)
- 이미 제거된 줄은 타겟에서 제외
- 남은 활성 줄이 2줄 이하면 제거 대신 "레이저 스윕" (잠깐 지나가고 복구)

### 6.4 레이저 패턴 변화 (후반)

| 시간대 | 패턴 |
|--------|------|
| 0~20초 | 단일 행 또는 열 |
| 20~40초 | 행/열 랜덤 + 가끔 2줄 동시 |
| 40~60초 | 2줄 동시 + 대각선 레이저 등장 |

---

## 7. 플레이어 시스템

### 7.1 이동

- 방향키(←↑↓→) 또는 WASD로 이동
- 한 번에 1칸씩 이동
- 빈 칸(제거된 셀)이나 그리드 밖으로는 이동 불가
- 키를 누르고 있으면 일정 간격으로 연속 이동 (150ms 딜레이)

### 7.2 모바일 조작

- 스와이프 방향으로 이동
- 또는 화면 하단 가상 D-패드 표시

### 7.3 시각 표현

- 플레이어: 밝은 네온 색상의 원형 마커
- 이동 시 잔상(trail) 이펙트
- 피격 시 폭발 파티클

## 8. Canvas 이펙트

### 8.1 레이저 빔 애니메이션

```
경고 단계 (1초)
  → 해당 줄 셀들이 붉게 깜빡임 (0.15초 주기)
  → 가장자리에서 레이저 조준선 표시

발사 단계 (0.3초)
  → 빔이 한쪽 끝에서 반대쪽으로 관통
  → 밝은 백색 → 붉은색 그라디언트
  → 셀 파괴 파티클 (파편 흩어짐)
  → 화면 전체 플래시 (0.05초)

잔여 효과 (0.5초)
  → 빔 자리에 잔열 라인 페이드아웃
  → 제거된 셀 가장자리 불꽃 잔여물
```

### 8.2 화면 흔들림 (Screen Shake)

- 레이저 발사 시 캔버스 전체가 2~4px 랜덤 오프셋으로 흔들림
- 0.3초간 지속, 감쇠(ease-out)

### 8.3 파티클 시스템

| 이벤트 | 파티클 |
|--------|--------|
| 셀 파괴 | 사각형 파편 8~12개, 중력 적용, 0.8초 수명 |
| 플레이어 피격 | 원형 폭발 20개, 방사형, 1초 수명 |
| 승리 | 화면 전체 골드 파티클 30개, 상승, 2초 수명 |
| 플레이어 이동 | 잔상 1~2개, 0.3초 페이드아웃 |

### 8.4 배경 효과

- 다크 배경에 미세한 그리드 패턴
- 시간 경과에 따라 배경색 점점 붉어짐 (긴장감)
- 가장자리에 비네팅 효과

## 11. IndexedDB 저장 데이터

### 11.1 스키마

```
Store: "records"
├── { id: "highscore", value: 48.2 }      // 최고 생존 시간 (초)
├── { id: "games", value: 27 }             // 총 플레이 횟수
├── { id: "wins", value: 3 }               // 승리 횟수 (60초 달성)
└── { id: "history", value: [              // 최근 10게임 기록
        { time: 34.7, date: "2026-04-07", won: false },
        { time: 60.0, date: "2026-04-07", won: true },
        ...
     ]}
```

### 11.2 기록 갱신 시점

| 이벤트 | 저장 항목 |
|--------|----------|
| 게임 오버 | games +1, history 추가, highscore 갱신 여부 확인 |
| 승리 | games +1, wins +1, history 추가, highscore 갱신 |
| 앱 진입 | highscore, games, wins 로드 → 타이틀에 표시 |

## 12. 기술 구현 상세

### 12.1 게임 루프

```
requestAnimationFrame 기반 60fps 루프

매 프레임:
  1. 입력 처리 (방향키 상태 확인)
  2. 게임 로직 업데이트
     - 플레이어 이동 (입력 쿨다운 적용)
     - 레이저 타이머 체크
     - 경고 상태 전환
     - 충돌 판정 (플레이어 vs 레이저)
     - 생존 시간 갱신
  3. 파티클 업데이트 (위치, 수명, 알파)
  4. 렌더링
     - 배경 + 그리드
     - 셀 상태별 색상
     - 레이저 빔 애니메이션
     - 플레이어
     - 파티클
     - 화면 흔들림 오프셋 적용
     - HUD 오버레이
```

### 12.2 레이저 발사 사이클

```
[대기] ─── interval 경과 ──→ [타겟 선택]
                                  │
                         플레이어 줄 제외
                         제거된 줄 제외
                         랜덤 행/열 선택
                                  │
                                  ▼
                            [경고 1초]
                          셀 깜빡임 시작
                                  │
                                  ▼
                            [발사 0.3초]
                          빔 애니메이션
                          셀 제거 + 파티클
                          화면 흔들림
                                  │
                           플레이어 피격?
                          ├─ Yes → 게임 오버
                          └─ No → [대기]로 복귀
                                  interval × 0.92
```

### 12.3 키 입력 처리

```
keydown → 키 상태 Map에 등록
keyup → 키 상태 Map에서 해제

매 프레임 입력 처리:
  - 마지막 이동 후 150ms 이상 경과했으면
  - 활성 방향키 중 가장 마지막에 누른 키 기준으로 이동
  - 이동 가능 여부 확인 (그리드 범위, 빈 칸 체크)
  - 이동 시 잔상 파티클 생성
```

## 13. 난이도 밸런스

### 13.1 생존 가능 셀 관리

- 64칸 중 레이저로 매 라운드 8칸(한 줄)씩 제거
- 최대 제거: 6줄(48칸) → 남은 2줄(16칸)에서 생존
- 줄이 2줄 이하로 남으면 "스윕 모드" → 제거 대신 잠깐 지나감
- 이를 통해 60초까지 물리적으로 생존 가능하도록 보장

### 13.2 체감 난이도 곡선

```
쉬움 ─────────────── 보통 ─────────── 어려움 ─── 극한
0s        10s        20s        30s     40s    50s   60s
│          │          │          │       │      │     │
여유롭게    슬슬       레이저     2줄    빠른    탈출  승리!
이동       긴장       빨라짐     동시    연속    불가능급
```
